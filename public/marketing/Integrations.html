How the sync works

One direction for data, no surprises.

Your HRIS is the source of truth. Bryte Rewards reads from it on a schedule you control, and never writes back unless you explicitly opt in.

  • Nightly sync by default. Can be set to real-time via webhooks.
  • Field-level mapping you can preview before anything goes live.
  • Soft-delete awareness: departed employees are archived, not erased, so their recognition history survives.
  • Manager and team changes are picked up within minutes — Monday-morning reorgs don't break Tuesday's feed.
  • Optional write-back: push anniversary badges & recognition milestones as custom fields into your HRIS.

Developer API

Or build your own, cleanly.

REST + webhooks, OpenAPI-documented, versioned. API access is included on Team & Enterprise. A typical first integration ships in an afternoon.

# Post a recognition from your internal tool
curl -X POST https://api.bryte.ca/v1/recognitions \
  -H "Authorization: Bearer $BRYTE_TOKEN" \
  -H "Content-Type: application/json" \
  -d '{
    "from_email": "nadia@acme.ca",
    "to_email": "marcus@acme.ca",
    "value_slug": "creative_courage",
    "message": "The Nordstrom pitch was yours from the first slide.",
    "points": 50
  }'

# Subscribe to webhooks
POST /v1/webhooks  → "recognition.sent", "badge.earned", "reward.redeemed"
